# RAG 智能知识平台 · 登录单页成功 UI 提示词（定稿）

> **性质**：依据实装落地的登录首页（`/login`）提炼的**可复用成功提示词**，供后续复刻、评审或交给设计/工程 AI。  
> **范围**：仅前端**登录单页**（品牌落地 + 登录浮窗）；不含管理后台 Layout / 业务页。  
> **工程边界**：Vue 3 + Vite + Element Plus + Pinia；主文件 `frontend/src/views/Login.vue`，可选 `styles/login.css`，Token 仅增 `--login-*`。  
> **视觉立场**：哑光深空、冷蓝强调、克制光效——企业级 AI 知识平台的「静谧科技」，而非娱乐向霓虹。  
> **定稿日期**：2026-07-18

## 一、提示词正文（可整段复制）

```text
请设计并描述一版「企业 RAG 智能知识平台」的登录单页高保真 UI。
这是产品的门面页：品牌优先、氛围完整、交互克制。最终实现为 Vue 3 + Element Plus，请输出设计说明与 Token，勿改业务契约。

━━━━━━━━━━━━━━━━
【0. 一句话气质】
━━━━━━━━━━━━━━━━
哑光纯黑底上的透视点阵深空；冷蓝为主交互色；中英混排无衬线；首屏只做「进门」——不堆仪表盘、不堆营销卡片。

━━━━━━━━━━━━━━━━
【1. 品牌双层结构（必须遵守）】
━━━━━━━━━━━━━━━━
· 提供方品牌（固定、管理员改不了）
  - 顶栏左：几何三方块 Logo + 名称「六朵云」
  - 页脚弱署名：Powered by 六朵云 · RAG Platform（不读后台 brand_footer_text）
· 使用方白标（读 branding 配置）
  - 主视觉大标题：brand_name（系统名称）
  - 登录弹窗：Logo / 欢迎语 / 主题色跟随 --el-color-primary
· 禁止把整页写死成某个客户名；禁止用衬线标题字体抢戏

━━━━━━━━━━━━━━━━
【2. 首屏构图（一个画面，一件事）】
━━━━━━━━━━━━━━━━
垂直居中、信息极简，自上而下仅允许：

① 状态徽章（小胶囊）
   - 文案：「知识即服务」
   - 左侧一枚主题色圆点微光
   - 半透底 + 极细浅描边，字号小于标题，字距略松

② 主标题（英雄级）
   - 内容：brand_name
   - 纯白、加粗、字号最大；字重明显高于副题

③ 副标题（同级字号、降明度）
   - 文案：「让企业知识即问即答」
   - 浅灰白（介于纯白与死灰之间），与主标题拉开主次，而非另起一段说明文

④ 唯一主 CTA
   - 文案：ENTER（全大写，无括号、无箭头）
   - 形态：全圆角胶囊（pill），非切角金属条、非双边按钮组
   - 填色：实心近白，深色字；相对标题区略下移，形成呼吸
   - 动效：仅外发光「脉冲呼吸」循环；禁止常驻蓝色描边环、禁止贴边跑马灯
   - 点击：打开登录浮窗（与 SIGN IN 同职）

禁止出现在首屏：
- 第二句产品说明（如「知识库智能管理·文档智能检索」）
- 双 CTA（如 Start + Demo）
- 统计条、日程、地址、贴纸、角标堆叠
- 首屏卡片墙

━━━━━━━━━━━━━━━━
【3. 顶栏结构】
━━━━━━━━━━━━━━━━
三栏栅格，贴顶固定，不挡主视觉点击（粒子层在下）：

左｜提供方「六朵云」
中｜四项浅灰导航（等分）：关于我们 · 知识服务 · 服务支持 · 联系我们
右｜SIGN IN（主题色字，字距偏松）

导航文字默认浅灰；hover / 激活：提亮近白 + 底部细线。
SIGN IN hover：主题色提亮 + 同款细线下划。

━━━━━━━━━━━━━━━━
【4. 导航下拉（局部浮层，非通栏 Mega）】
━━━━━━━━━━━━━━━━
交互准则：
· 以「点击」唤起；禁止仅靠移入打开
· 点同项再收起；点页面空白关闭；Esc 关闭
· 一次只展开一项对应内容

形态准则：
· 下拉卡片出现在「该导航项正下方」，相对导航水平居中对齐
· 毛玻璃卡片向上延展，覆盖该导航标题所在区域，使标题视觉落入卡片顶部（避免标题与卡片各画一层、重复标题）
· 卡片：深色半透渐变 + 中等高斯模糊 + 细淡蓝描边 + 大圆角 + 底部浮空软阴影；粒子底可柔和透出
· 子选项垂直排布、居中；默认浅灰
· 子选项 hover / 选中：与顶栏一致——提亮 +「主题色蓝」细下划线；禁止灰底方框选中态

子链为占位文案（可替换），结构示例：
· 关于我们 → 平台愿景 / 技术理念 / 交付能力 / 白标定制
· 知识服务 → 知识库管理 / 文档智能检索 / 企业级问答 / 命中率验证
· 服务支持 → 部署上线 / 权限与角色 / 运维保障 / 授权申请
· 联系我们 → 商务合作 / 技术支持 / 演示预约 / 工作时间

━━━━━━━━━━━━━━━━
【5. 氛围底（粒子场）】
━━━━━━━━━━━━━━━━
· 全屏 Canvas：透视网格点阵 + 共享高度场起伏，作环境底，不作主角
· 哑光近黑基底；点为冷白/淡灰，忌高饱和彩虹粒子
· 可选：指针邻近区域柔和冷蓝提亮（聚焦），不破坏波场结构
· 主文案与 CTA 必须始终压在粒子之上，对比可读

━━━━━━━━━━━━━━━━
【6. 登录浮窗】
━━━━━━━━━━━━━━━━
· SIGN IN / ENTER 打开；点遮罩或关闭钮收起
· 高透明雾化毛玻璃：上深下泛冷蓝的垂直渐变；细蓝描边；大圆角
· 内含：白标 Logo、欢迎标题、员工/管理员切换、用户名密码（密码眼常显）、错误与「忘记密码？」同行、主登录按钮
· 错误提示在弹窗内可见，勿被遮罩压住
· 不透明度略高于早期「过透」版本，保证表单可读，仍透出底图粒子

━━━━━━━━━━━━━━━━
【7. 字体 · 色彩 · Token】
━━━━━━━━━━━━━━━━
字体：
· 中文：Noto Sans SC；西文：DM Sans（或同等现代无衬线）
· 全页统一无衬线；禁止标题改宋体/衬线「文艺海报」风

色彩：
· 页底：#000 近纯黑哑光
· 主文：近白；辅文：中灰；更弱说明：低明度灰
· 强调与下划线、光点：var(--el-color-primary)（品牌可配，现多为冷蓝）
· 忌：紫粉渐变网红风、暖奶油底 + 陶土强调、报纸排版风、大面积霓虹描边

Token 建议（登录页专用，只增不改名）：
--login-bg / --login-text / --login-text-muted / --login-text-dim
--login-nav-hover / --login-surface / --login-border / --login-glow
--login-font-sans

━━━━━━━━━━━━━━━━
【8. 动效预算（克制）】
━━━━━━━━━━━━━━━━
至少保留、且仅保留有目的的动效：
1. 粒子场持续缓慢流动
2. ENTER 外发光脉冲呼吸
3. 下拉 / 弹窗：短距位移动画 + 透明度缓动（约 0.2–0.35s）

禁止：常驻电光描边轮转、多重阴影叠噪、无意义弹跳。

━━━━━━━━━━━━━━━━
【9. 工程与协作硬约束】
━━━━━━━━━━━━━━━━
· 只改登录侧：Login.vue（及 login.css、--login-*）
· 禁止改：router path、api 契约、Pinia 业务状态机、权限工具
· 登录成功跳转逻辑保持：管理员 → /dashboard；员工 → /chat
· 响应式：窄屏顶栏可换行缩放，四项导航与 SIGN IN 不可无故隐藏

请输出：① 首屏与顶栏结构说明 ② Token 表 ③ 关键态列表（hover/active/打开）④ 与上述约束的符合性自检清单。
```

## 三、评审用「成功判据」（一眼可验）

1. **品牌测试**：隐去顶栏后，首屏仍能通过大标题识别「这是哪套产品/租户」，而非通用模板站。  
2. **主次测试**：主标题纯白最重；副题明显退一阶；徽章最轻。  
3. **CTA 测试**：全页只有一颗主行动点 ENTER（SIGN IN 为同职次入口）。  
4. **光效测试**：ENTER 有呼吸、无常驻蓝环。  
5. **导航测试**：点击才出层；层在对应项下居中；子链蓝下划线。  
6. **克制测试**：无双按钮、无首屏说明长文、无贴纸角标墙。
