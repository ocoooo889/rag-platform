## R01-R16 风险项及应对预案

### R01：API Key 余额不足或失效

| 属性 | 内容 |
|------|------|
| **编号** | R01 |
| **级别** | 🔴 P0 |
| **风险描述** | LLM/Embedding API Key 余额不足或被封禁，导致 RAG 核心功能完全不可用 |
| **触发场景** | ① Key 余额耗尽 ② Key 权限被限制 ③ Key 被误删 |
| **影响范围** | 向量检索、智能对话、命中测试全链路 |

**应对预案：**

1. **预防措施**：
   - 开工前确认全员 Key 余额 ≥ 100 元
   - 每人使用独立 Key，避免共用限流
   - 监控 Key 消耗速率，设置余额告警

2. **应急响应**：
   - 切换备用 Key（负责人准备 2-3 个备用 Key）
   - 启用降级模式：Embedding 不可用时切换为纯 BM25 检索
   - LLM 不可用时返回兜底文案

3. **恢复流程**：
   - 充值或更换 Key 后恢复正常模式
   - 验证全链路功能正常

### R02：Chroma 向量库服务不可用

| 属性 | 内容 |
|------|------|
| **编号** | R02 |
| **级别** | 🔴 P0 |
| **风险描述** | Chroma HTTP 服务启动失败、端口占用或网络不通，导致向量检索完全不可用 |
| **触发场景** | ① 端口 8000 被占用 ② Chroma 进程崩溃 ③ 防火墙拦截 |
| **影响范围** | 向量检索、命中测试、智能对话 |

**应对预案：**

1. **预防措施**：
   - 启动前检查端口占用：`netstat -ano | findstr :8000`（Windows）/ `lsof -i :8000`（Mac/Linux）
   - 使用 `scripts/start_all.sh/bat` 统一启动

2. **应急响应**：
   - 释放端口：杀死占用进程
   - 重启 Chroma：`chroma run --path ./chroma_data --port 8000`
   - 启用降级模式：切换为纯 BM25 全文检索

3. **恢复流程**：
   - 验证 Chroma 连接正常：访问 `http://localhost:8000/api/v2/heartbeat`（勿用已弃用的 `/api/v1/`）
   - 重建向量索引（如需）

### R03：Embedding 调用超时/失败

| 属性 | 内容 |
|------|------|
| **编号** | R03 |
| **级别** | 🟠 P1 |
| **风险描述** | Embedding API 响应超时或返回错误，导致文档向量化失败 |
| **触发场景** | ① API 服务拥堵 ② 网络延迟 ③ 模型版本变更 |
| **影响范围** | 文档上传、向量化、向量检索 |

**应对预案：**

1. **预防措施**：
   - 设置超时时间：`EMBEDDING_TIMEOUT = 10`（秒）
   - 配置重试机制：`LLM_MAX_RETRIES = 1`

2. **应急响应**：
   - 自动降级：关闭向量检索，仅保留 BM25 全文检索
   - 前端提示："向量模型暂不可用，已切换关键词检索"

3. **恢复流程**：
   - 检查 API 状态，确认恢复后重启向量检索
   - 重新处理未完成的文档

### R05：文档处理失败（切片/向量化异常）

| 属性 | 内容 |
|------|------|
| **编号** | R05 |
| **级别** | 🟠 P1 |
| **风险描述** | 文档上传后处理失败，状态停留在 `processing` 或变为 `failed` |
| **触发场景** | ① 文件格式不支持 ② 编码问题 ③ Embedding 调用失败 |
| **影响范围** | 文档检索、命中测试 |

**应对预案：**

1. **预防措施**：
   - 前端限制上传格式：仅支持 `.md`、`.txt`、`.pdf`
   - 后端增加文件校验和异常捕获

2. **应急响应**：
   - 文档状态标记为 `failed`，前端显示错误提示
   - 用户可重新上传文件

3. **恢复流程**：
   - 排查具体失败原因（查看日志）
   - 修复问题后支持重新处理失败文档

### R06：用户组权限越权访问

| 属性 | 内容 |
|------|------|
| **编号** | R06 |
| **级别** | 🔴 P0 |
| **风险描述** | 用户可访问未授权的用户组下的知识库和文档，违反数据隔离要求 |
| **触发场景** | ① 权限中间件逻辑错误 ② SQL 查询未加用户组过滤 ③ 接口漏加权限校验 |
| **影响范围** | 数据安全、权限隔离 |

**应对预案：**

1. **预防措施**：
   - 所有 KB/文档接口必须加用户组过滤
   - admin 角色可查看所有资源（超管模式）
   - 未分配用户组的用户返回 4001 错误

2. **应急响应**：
   - 立即修复权限中间件和 SQL 查询
   - 增加接口级别的权限校验

3. **恢复流程**：
   - 执行权限越权专项测试用例
   - 验证跨组访问被正确阻断

### R07：文档状态准入校验失效

| 属性 | 内容 |
|------|------|
| **编号** | R07 |
| **级别** | 🟠 P1 |
| **风险描述** | 未处理完成的文档（状态为 `pending`/`processing`）仍可被检索，导致检索结果不准确 |
| **触发场景** | ① 检索接口未校验文档状态 ② 状态轮询机制失效 ③ 前端未禁用未完成文档 |
| **影响范围** | 命中测试、智能对话 |

**应对预案：**

1. **预防措施**：
   - 检索接口强制校验文档状态，非 `completed` 返回 4002
   - 前端命中测试下拉中禁用未完成文档并显示提示

2. **应急响应**：
   - 修复后端状态校验逻辑
   - 修复前端禁用逻辑

3. **恢复流程**：
   - 执行文档状态准入专项测试用例
   - 验证未完成文档不可检索

### R08：混合检索融合公式错误

| 属性 | 内容 |
|------|------|
| **编号** | R08 |
| **级别** | 🟡 P2 |
| **风险描述** | 混合检索权重计算错误，导致检索结果质量下降 |
| **触发场景** | ① alpha 参数配置错误 ② BM25 得分未归一化 ③ 公式实现错误 |
| **影响范围** | 命中测试、智能对话检索效果 |

**应对预案：**

1. **预防措施**：
   - 写死公式：`score = alpha * vector_score + (1 - alpha) * bm25_score`
   - alpha 固定为 0.7，不可修改

2. **应急响应**：
   - 检查公式实现，修复计算逻辑
   - 验证检索结果排序正确

3. **恢复流程**：
   - 使用标准测试问题集验证检索效果
   - 确认命中数和相关性符合预期

### R10：前端代理配置错误

| 属性 | 内容 |
|------|------|
| **编号** | R10 |
| **级别** | 🟠 P1 |
| **风险描述** | `vite.config.js` 代理端口配置错误，导致前端 API 请求全部失败 |
| **触发场景** | ① 代理 target 端口与后端实际端口不一致 ② 域名配置错误 |
| **影响范围** | 前端所有接口调用 |

**应对预案：**

1. **预防措施**：
   - 后端固定端口：`API_PORT = 8001`
   - 代理配置：`target: 'http://127.0.0.1:8001'`
   - 启动前检查端口一致性

2. **应急响应**：
   - 修改 `vite.config.js` 代理配置，重启前端服务

3. **恢复流程**：
   - 验证前端接口调用正常
   - 确认所有页面数据可正常加载
