## 六、命中率测试（TC-H）— 9个用例

| 编号 | 测试场景 | 操作步骤 | 预期结果 |
|:--:|------|------|------|
| TC-H01 | 未选择文档 | 知识库下拉已选，文档下拉留空，点击"运行测试" | 前端拦截，提示"请选择目标文档"，不发起请求 |
| TC-H02 | 输入空问题 | 知识库+文档已选，问题输入框留空，点击"运行测试" | 前端拦截，提示"请输入测试问题"，不发起请求 |
| TC-H03 | 检索无匹配 | 选择文档，输入一个与该文档内容完全无关的问题 | code:0，total_hits:0，hits:[]，前端展示空状态提示"未检索到相关内容" |
| TC-H04 | 三种检索模式差异化 | 同一文档、同一问题，分别切换 vector / keyword / hybrid 三种方式各运行一次 | 三次返回结果排序不同：vector 偏语义匹配、keyword 偏关键词匹配、hybrid 融合排序 |
| TC-H05 | 跨知识库隔离 | 知识库A中有文档doc1，知识库B中无文档。选择知识库B | 知识库B的下拉文档列表中不出现doc1，无法跨库检索 |
| TC-H06 | 文档未处理完成 | 上传一篇文档后立即在命中测试中选择该文档（status=processing），点击"运行测试" | 前端弹出等待提示"文档正在处理中，请等待处理完成后再试"；后端返回 code:4002 |
| TC-H07 | 文档处理失败 | 将一篇文档状态标记为 failed，在命中测试中选择该文档，点击"运行测试" | 前端弹出提示"文档处理失败，请重新上传或联系管理员"；下拉中该文档标注红色"失败"标签且禁用 |
| TC-H08 | 跨用户组越权访问 | 用户A属于用户组G1（授权KB1），用户B属于用户组G2（授权KB2）。用户A尝试通过API直接访问KB2的文档 | 返回 code:403，提示"您无权访问该知识库"；知识库列表中不出现KB2 |
| TC-H09 | 未分配用户组 | 新建用户但不分配任何用户组，登录后查看知识库列表 | 知识库列表为空，返回 code:4001，"您尚未被分配到任何用户组，请联系管理员" |

## 七、智能对话（TC-C）— 3个用例

| 编号 | 测试场景 | 操作步骤 | 预期结果 |
|:--:|------|------|------|
| TC-C01 | 正常对话 | POST `/api/chat/send`，body: `{"kb_id":"kb_xxx","query":"年假有几天？","search_type":"hybrid"}` | code:0，返回 answer 和 references，session_id 自动生成，references 含 chunk_id/content/score |
| TC-C02 | 多轮对话 | 第一次对话获取 session_id，第二次对话传入相同 session_id + 新 query | code:0，LLM 回答基于上下文（含历史），对话历史被正确加载 |
| TC-C03 | 流式对话（SSE） | POST `/api/chat/stream`，body 同 TC-C01 | Content-Type: text/event-stream，依次收到 type:start → type:chunk（多次）→ type:done（含 references） |
