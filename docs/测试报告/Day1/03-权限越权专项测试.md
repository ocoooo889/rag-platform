## 测试用例

### TC-P01：用户组内正常访问

**目的**：验证用户组权限正常生效——用户只能看到自己组内被授权的知识库。

**前置条件**：
- user_a 属于 G1，G1 被授权 KB1
- KB1 下有 1 篇已完成文档

**操作步骤**：

| 步骤 | 操作 | 预期结果 |
|:--:|------|------|
| 1 | 以 user_a 登录 | 登录成功，跳转到知识库管理页 |
| 2 | 进入知识库管理页 | 知识库列表仅显示 KB1 |
| 3 | 调用 `GET /api/knowledge-bases` 接口 | 返回的 data 数组中仅包含 KB1，不包含 KB2 |

**验证点**：
- ✅ 知识库列表仅显示 KB1
- ✅ KB2 不出现在列表和下拉选项中
- ✅ 响应中不包含任何 KB2 的信息

---

### TC-P02：跨组访问知识库文档列表

**目的**：验证用户不能通过直接调用 API 获取其他用户组的文档列表。

**前置条件**：同 TC-P01。

**操作步骤**：

| 步骤 | 操作 | 预期结果 |
|:--:|------|------|
| 1 | 以 user_a 登录，获取 Token | 获取到 Token |
| 2 | 调用 `GET /api/knowledge-bases/{KB2_id}/documents`，携带 user_a 的 Token | HTTP 状态码 403 |
| 3 | 检查响应体 | `code: 403, msg: "您无权访问该知识库", data: null` |

**验证点**：
- ✅ HTTP 状态码为 403
- ✅ 响应体 code 为 403
- ✅ 不返回任何文档数据

---

### TC-P03：跨组命中测试

**目的**：验证用户不能对其他用户组的文档执行命中率测试。

**前置条件**：
- user_a 属于 G1，KB2 属于 G2
- KB2 下有已完成文档 doc_m1

**操作步骤**：

| 步骤 | 操作 | 预期结果 |
|:--:|------|------|
| 1 | 以 user_a 登录，获取 Token | 获取到 Token |
| 2 | 调用 `POST /api/rag/test_retrieve`，请求体：`{"kb_id": "KB2的ID", "doc_id": "doc_m1的ID", "search_type": "hybrid", "query": "测试问题"}` | HTTP 状态码 403 |
| 3 | 检查响应体 | `code: 403, msg: "您无权访问该知识库", data: null` |

**验证点**：
- ✅ 接口在权限校验阶段就返回 403
- ✅ 不会走到检索逻辑（不会返回检索结果或 4002）

---

### TC-P04：跨组智能对话

**目的**：验证用户不能在其他用户组的知识库范围内进行智能对话。

**前置条件**：同 TC-P03。

**操作步骤**：

| 步骤 | 操作 | 预期结果 |
|:--:|------|------|
| 1 | 以 user_a 登录，获取 Token | 获取到 Token |
| 2 | 调用 `POST /api/chat/send`，请求体：`{"kb_id": "KB2的ID", "query": "测试问题"}` | HTTP 状态码 403 |
| 3 | 检查响应体 | `code: 403, msg: "您无权访问该知识库", data: null` |
| 4 | 调用 `POST /api/chat/stream`，请求体：`{"kb_id": "KB2的ID", "query": "测试问题"}` | HTTP 状态码 403 |

**验证点**：
- ✅ `/send` 和 `/stream` 两个接口都返回 403
- ✅ 不会进入 LLM 调用逻辑
- ✅ 不会消耗 Token

### TC-P05：未分配用户组

**目的**：验证未分配用户组的用户无法看到任何知识库，返回 4001。

**前置条件**：
- user_c 不属于任何用户组
- 系统中存在 KB1、KB2 两个知识库

**操作步骤**：

| 步骤 | 操作 | 预期结果 |
|:--:|------|------|
| 1 | 以 user_c 登录 | 登录成功 |
| 2 | 进入知识库管理页 | 列表为空，提示"您尚未被分配到任何用户组" |
| 3 | 调用 `GET /api/knowledge-bases` | `code: 4001, msg: "您尚未被分配到任何用户组，请联系管理员", data: null` |
| 4 | 调用 `POST /api/rag/test_retrieve`，传任意 kb_id | `code: 4001` |
| 5 | 调用 `POST /api/chat/send`，传任意 kb_id | `code: 4001` |

**验证点**：
- ✅ 知识库列表接口返回 4001（不是 403）
- ✅ 4001 和 403 有明确区分：4001 是"未分配用户组"，403 是"无权访问该资源"
- ✅ 命中测试和对话接口同样返回 4001

## 异常码速查

| code | 场景 | 含义 |
|:--:|------|------|
| 0 | 正常访问 | 用户有权限访问该资源 |
| 401 | 未登录/Token 失效 | 未携带 Token 或 Token 校验失败 |
| 403 | 权限不足 | 已登录但无权访问该资源（跨组访问） |
| 4001 | 未分配用户组 | 用户不属于任何用户组 |
