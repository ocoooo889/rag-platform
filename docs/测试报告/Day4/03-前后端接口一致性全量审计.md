# Day4：前后端接口一致性全量审计

> **日期**：2026-07-19  
> **依据**：运行中后端 `GET /openapi.json`（45 条）+ `frontend/src/api/*` + 实请求抽检  
> **环境**：`zhangyuday4-fix` / 本机 `127.0.0.1:8001`  

---

## 一、结论（先看这三句）

1. **主链路路径与方法整体一致**：登录、知识库、文档、命中测试、会话流式、Dashboard、品牌、用户/角色/用户组/模型等，前端调用的路径与 OpenAPI **对齐**。  
2. **字段名关键点正确**：对话 / 命中测试均使用 **`query`**（不是 `question`）；代理指向 **8001**（不是 Chroma 8000）。  
3. **存在少量不一致 / 死代码**，不阻塞主链路，但建议清掉（见 §三）。

## 三、不一致与风险清单

| 级别 | 问题 | 说明 | 建议 |
|:----:|------|------|------|
| P1 | 会话 fallback 死路径 | `chat.js` 在主路径失败时会再试 `GET/DELETE /api/chat/{id}/…`，OpenAPI **无此路由**（实请求 404） | 删除 fallback，只保留 `/sessions/{id}` |
| P2 | `userStore.login()` JSON 登录 | 与真实页 `loginApi`（form）重复；若误用可能失败 | 删掉或改为调用 `loginApi` |
| P2 | 角色 `description` | 前端创建角色带 `description`，后端 `RoleCreate` **无该字段**（Pydantic 忽略多余字段，创建仍成功） | 前端去掉，或后端 schema 增加可选字段 |
| P2 | profile `avatar_url` | `updateProfileApi` 可传 `avatar_url`，后端 `ProfileUpdateRequest` **不接收**（头像仅 `POST /auth/avatar`） | 前端勿靠 profile 改头像（当前 Layout 已只改 display_name） |
| Info | chat `search_type` | 前端默认 **`vector`**；后端 schema 默认 **`hybrid`** | 有意优化首 token；页面传了则以后端收到值为准 |
| Info | `POST /api/chat/send` | 后端有、前端未接 | 保留作调试/兼容即可 |
| Info | `createChatSession` | 无后端创建接口；本地生成 id，首条 stream 落库 | 与后端设计一致 |
