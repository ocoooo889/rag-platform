# Day4：长文档向量化失败修复说明

> **日期**：2026-07-19  
> **现象**：长文档上传后无法正常向量检索；日志出现 `Embedding Connection error`，文档却显示「就绪」  

---

## 根因

1. Embedding 仅重试 **2 次**、超时偏短，长文档数百批调用易中途 **Connection error**  
2. 全量算完向量后 **一次** `collection.add`，大文档易超时/失败  
3. 向量失败仍标 `completed` + 警告「已切换关键词检索」，UI 看起来像成功  
4. 切片阶段仍为 `pending`，长文档长时间无进度反馈  

---

## 已做修复

| 项 | 变更 |
|----|------|
| Embedding | 默认超时 60s+；`EMBEDDING_MAX_RETRIES=4`；连接错误指数退避；批次进度日志 |
| OpenAI 客户端 | 加大 `timeout`，关闭 SDK 自带 max_retries（由业务重试） |
| Chroma | 按 `CHROMA_WRITE_BATCH_SIZE`（默认 100）分批写入 |
| 切片 | utf-8 / gbk 编码回退；`CHUNK_SIZE` 可用环境变量调整 |
| 状态 | 入库一开始即 `processing` |

本机 `.env` 建议：

```env
EMBEDDING_TIMEOUT=90
EMBEDDING_BATCH_SIZE=10
EMBEDDING_MAX_RETRIES=4
CHROMA_WRITE_BATCH_SIZE=100
```

---

## 使用注意

1. 上传长文档时 **不要** 用会频繁 `--reload` 的方式打断后台任务  
2. 已失败/仅关键词的文档请 **删除后重新上传**（或等后续提供重处理接口）  
3. 看后端日志是否有 `Embedding 进度 x/y 批`、`Chroma 写入进度`  
4. 文档列表若 `error_message` 仍含「向量模型暂不可用」，说明向量未写入成功  

---

## 验证建议

1. 重启后端使配置生效  
2. 上传一篇较长 `.md`（如数百 KB～数 MB）  
3. 等待状态从 processing → completed，且 `error_message` 为空  
4. 命中测试 / 智能对话使用 **vector**，确认能命中新文档内容
