### 方案 B — 登录主口 + `/me`（决策 3）

**事实**：前端 `auth.js` 调用 `POST /api/auth/login`（form）→ `login_form`。

#### B1. `login_form`（主修）

统一返回：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "access_token": "...",
    "token_type": "bearer",
    "user": {
      "id": 1,
      "username": "admin",
      "display_name": "...",
      "role": "admin",
      "role_name": "管理员",
      "status": "启用"
    }
  }
}
```

失败：`code` 401/403 + `msg`，`data: null`（与业务体一致）。  
`role_name`：`admin`→管理员，`user`→普通用户。

#### B2. `login_json` 同构

`ResponseModel`，`data` 字段与 B1 相同。

#### B3. `GET /api/auth/me`

需登录；返回与 `user` 对象同结构。依赖 `get_current_user`（须已 joinedload role，见方案 I）。

#### B4. `frontend/src/api/auth.js`

- 去掉 `role_name: '管理员'` 兜底。  
- 无 `user` 则抛错。  
- 继续走 form；`unwrap` 兼容 `{code,data}`。

#### B5. `Login.vue` + `user` store + 路由守卫

- 使用 `data.user.role`（code）判断管理员入口；普通用户默认进 `/chat` 或 `/knowledge-bases`（二选一，推荐 `/chat` 亦可，但菜单须仍能进 KB/命中）。  
- 有 token 无完整 userInfo 时：调 `/api/auth/me` 补齐（Layout `onMounted` 或 `router.beforeEach`）。

#### B6. 风险说明（Swagger）

`/login` 改为业务包裹后，Swagger「Authorize」OAuth2 密码流可能失效；**以前端联调为准**。若需兼容 Swagger，可另留纯 OAuth2 调试口（非达交必须）。

### 方案 E — 统一异常码（可执行唯一路径）

#### E1. 新增 `backend/app/utils/exceptions.py`

```python
class BusinessException(Exception):
    def __init__(self, code: int, msg: str, http_status: int = 200):
        self.code = code
        self.msg = msg
        self.http_status = http_status
```

**禁止** `permission.py` / `auth.py` `from app.main import ...`（防循环导入）。

#### E2. `main.py` 注册 handler

- `BusinessException` → `JSONResponse(http_status, {code, msg, data: null})`  
- `HTTPException` → 同上（`code` 用 HTTP status 或约定映射；`detail` 为 str 时作 `msg`）

#### E3. `permission.require_kb_access`

```python
raise BusinessException(code=403, msg="您无权访问该知识库", http_status=403)
```

**不要** `return JSONResponse`（避免与 `await require_kb_access(...)` 调用方语义混乱）。

#### E4. 前端

`request.js` 已覆盖 HTTP + 业务码双路径则无需大改；联调验证 403 提示文案正确即可。

### 方案 F — 会话 API（先修 create，再开开关）

#### F1. `frontend/src/api/chat.js` — `createChatSession`

## 六、达交签字条件（必须全部 ✅）

1. **身份真实**：登录返回 `role`/`role_name`；无硬编码管理员；`/me` 可刷新身份。  
2. **菜单按手册**：admin 见管理+RAG；user 见 KB/文档/命中/对话，**不见**用户/角色/用户组/品牌等管理页。  
3. **写/管理读鉴权**：非 admin 调管理 API → 业务码 403。  
4. **用户组隔离**：经用户组「成员+KB 授权」配置后，user 仅见授权 KB；未授权 403；未分组列表 4001（若适用）。  
5. **白标闭环**：改 `brand_name`/标语/页脚后，Login+Layout（及 favicon/主题若已接）生效；`/uploads` 在 dev 代理下可访问。  
6. **文档轮询**：进页有处理中文档或上传后，状态自动刷到终态；离开页停止 timer。  
7. **会话**：列表/消息/删除可走后端；新建不打错误 create 端点。  
8. **主链路无 Mock**：默认不靠静默假数据；登录→建库→上传→完成→命中→对话真实 API。  
9. **SSE 停止**：停止后 UI 正常收尾，不卡「思考中」。  
10. **删 KB**：关联向量不可再被检索命中。
