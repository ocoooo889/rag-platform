## 七、风险与「有条件通过」底线


| 风险                 | 影响                  | 建议降级                                    |
| ------------------ | ------------------- | --------------------------------------- |
| main 午间仍未挂 B       | CP3 必挂              | 负责人强制合入或临时发布分支                          |
| 无 API Key / Chroma | 文档长期 failed，对话 5002 | CP3 可演示 keyword 检索 + 4002/403；问答标「环境受限」 |
| 前端 B 仍占位           | 无法 UI 演示命中/对话       | 用 Swagger/脚本演示后端；UI 记 P0 遗留             |
| A 未加鉴权             | 跨组 rag/chat 可绕过     | CP3 仅认 KB 列表/文档列表隔离；P03/P04 不签字         |


**测试工程师对 CP3 的建议签字条件**：

- **必须**：正式入口可登录；KB/用户组隔离（P01/P02/P05/P06）；白标 GET/PUT；文档 pending→4002（后端）。  
- **应当**：三种检索至少 keyword 成功；chat 在有 Key 时成功或明确 5002 降级。  
- **可以遗留到收尾**：前端 B 完整 UI、Langfuse 大盘、Prometheus 业务计数 inc。
