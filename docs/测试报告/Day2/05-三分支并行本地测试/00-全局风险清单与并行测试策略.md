## A. 全局风险清单（多分支并行 / 合入）

| ID | 风险 | 涉及 | 级别 | 说明与规避 |
|---|---|---|---|---|
| G01 | **API 端口冲突** | 全员 | 高 | 默认都想占 `8001`。并行必须：shang=`8011`，luoyue-backend=`8012`，yulin=`8013` |
| G02 | **Chroma 端口/数据冲突** | A 侧 | 高 | 默认 `CHROMA_PORT=8000`。可：① 共用一台 Chroma 但 **Collection 后缀隔离**；② 或分别 `8000/8010/8020` + 不同 `--path` |
| G03 | **SQLite 库文件覆盖** | 全员 | 高 | 默认相对路径 `./{ENV}_rag.db`。必须每人 `LOCAL_DB_NAME` 不同，且在各自 worktree 根目录生成 |
| G04 | **UPLOAD_DIR 互踩** | docs/branding | 高 | 设 `UPLOAD_DIR=./uploads/<person>` |
| G05 | **前端端口 5173 冲突** | luoyue + 其他前端 | 中 | `npm run dev -- --port 5174/5175` |
| G06 | **Vite 代理误指向 8000** | luoyue | 高 | 8000=Chroma。代理必须指向对应后端端口（801x） |
| G07 | **Mock 假绿** | luoyue | 高 | 默认 Mock on；并行联调务必关 Mock |
| G08 | **A/B 接口契约分裂** | shang↔luoyue | 高 | `doc_id` vs `doc_ids`，`search_type` vs `mode`，`hits` vs `items` |
| G09 | **yulin 双 prefix** | yulin↔前端/契约 | P0 | 实际路径变成 `/api/xxx/api/xxx/...`，标准路径 404 |
| G10 | **yulin ORM≠init_db** | yulin | P0 | `hashed_password`/`created_at`/`system_config`/`admin` 名不一致 → 500 |
| G11 | **空 api 文件合入覆盖** | shang/luoyue → yulin | P0 | shang/luoyue 的 `auth.py` 等为 0 字节；merge 时禁止用空文件覆盖 yulin |
| G12 | **chat.py / llm_client 多分支同改** | shang↔yulin | 高 | 合入前先 rebase/对齐，避免静默丢逻辑 |
| G13 | **依赖未写入 requirements** | yulin | 高 | `prometheus-fastapi-instrumentator`、`loguru` 需写入，否则干净环境起不来 |
| G14 | **共用 conda 环境版本漂移** | 全员 | 中 | 并行测试强制 **每 worktree 独立 venv** |
| G15 | **A 无鉴权 vs B 强制 JWT** | 合入后 | 高 | 合并后权限模型需统一，否则越权或前端全挂 |
| G16 | **LLM Key / 限流** | chat | 中 | 每人独立 `OPENAI_API_KEY`；并行压测易触发 429/超时 |
| G17 | **init_db 启动反复执行** | yulin | 中 | 错 schema 被「IF NOT EXISTS」锁死，需删库重建 |

### 合入优先级建议（测试视角）

1. **先修 yulin**：去双 prefix + 统一 schema（否则 B 无法测）  
2. **再对齐契约**：前端 luoyue ↔ 后端 A（shang）字段  
3. **最后合 main**：保留 yulin 非空 api 文件，解决 chat 冲突，补鉴权

### B4. 并行时「禁止事项」

1. 不要三人共用同一个 `*.db` 文件名  
2. 不要同时 `uvicorn` 占同一端口  
3. 不要在未改 proxy 时对 luoyue 关 Mock  
4. 不要把 shang/luoyue 的 **空** `backend/app/api/*.py` merge 进 yulin  
5. yulin schema 未修好前，不要用同一 DB 文件来回切换 ORM/init_db（先删 `dev_yulin_rag.db` 再测）
