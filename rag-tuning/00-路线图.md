# RAG 调优路线图

## 顺序（严格按此）

```text
第 0 步 评测基线
   ↓
第 1 步 Query Rewrite（多轮）
   ↓
第 2 步 jieba BM25
   ↓
第 3 步 大库候选缩小 / 调 α（有数据再动）
   ↓

```

## 第 0 步 — 评测基线

**目标：** 留下「改之前」的分数，否则无法证明调优有效。

**你要做的：**

1. 确认环境（见 README）
2. 填 [01-评测基线表.md](./01-评测基线表.md) 的黄金 10 问
3. 至少跑 [02-加压难问集.md](./02-加压难问集.md) 里标了 ★ 的 5 条
4. 可选：`python rag-tuning/scripts/run_baseline.py` 自动打 keyword/vector/hybrid

**完成标志：** 基线表里「改前」列填完，并复制一份到 `results/baseline-YYYYMMDD.md`
