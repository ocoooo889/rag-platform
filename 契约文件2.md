## 一、V2 新增数据表定义

### 1.1 user_groups - 用户组表

| 字段 | 类型 | 约束 | 说明 |
|------|------|:----:|------|
| id | TEXT | PK, UUID | 用户组唯一标识 |
| name | TEXT | NOT NULL, UNIQUE | 用户组名称 |
| description | TEXT | DEFAULT '' | 组描述 |
| created_at | TEXT | NOT NULL | ISO 8601 时间戳 |

索引: idx_ug_name ON name

### 1.2 user_group_members - 用户-用户组关联表

| 字段 | 类型 | 约束 | 说明 |
|------|------|:----:|------|
| id | TEXT | PK, UUID | 关联唯一标识 |
| user_id | TEXT | FK -> users(id), NOT NULL | 用户 ID |
| group_id | TEXT | FK -> user_groups(id), NOT NULL | 用户组 ID |
| created_at | TEXT | NOT NULL | ISO 8601 时间戳 |

约束: UNIQUE(user_id, group_id)
外键: user_id -> users(id) ON DELETE CASCADE
外键: group_id -> user_groups(id) ON DELETE CASCADE

### 1.3 kb_group_access - 知识库-用户组授权表

| 字段 | 类型 | 约束 | 说明 |
|------|------|:----:|------|
| id | TEXT | PK, UUID | 授权唯一标识 |
| kb_id | TEXT | FK -> knowledge_bases(id), NOT NULL | 知识库 ID |
| group_id | TEXT | FK -> user_groups(id), NOT NULL | 用户组 ID |
| created_at | TEXT | NOT NULL | ISO 8601 时间戳 |

约束: UNIQUE(kb_id, group_id)
外键: kb_id -> knowledge_bases(id) ON DELETE CASCADE
外键: group_id -> user_groups(id) ON DELETE CASCADE

### 1.4 system_config - 系统配置表（白标/OEM）

| 字段 | 类型 | 约束 | 说明 |
|------|------|:----:|------|
| id | TEXT | PK, UUID | 配置唯一标识 |
| config_key | TEXT | NOT NULL, UNIQUE | 配置键 |
| config_value | TEXT | NOT NULL | 配置值 |
| config_type | TEXT | DEFAULT 'string' | string/color/url/json |
| updated_by | TEXT | FK -> users(id) | 最后修改人 |
| updated_at | TEXT | NOT NULL | ISO 8601 时间戳 |

种子数据:

| config_key | config_value | config_type | 说明 |
|------------|-------------|:-----------:|------|
| branding_system_name | 智能 RAG 平台 | string | 系统名称 |
| branding_logo_url | (空) | url | Logo 访问路径 |
| branding_theme_color | #409EFF | color | 主题色 |
| branding_login_slogan | AI 驱动的企业知识库 | string | 登录页标语 |
| branding_footer_text | Powered by RAG Platform | string | 页脚文案 |

## 二、V2 新增 API 接口契约

### 2.1 用户组管理（后端 B）

#### GET /api/user-groups - 用户组列表

查询参数: ?page=1&page_size=10

成功响应:
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [
      {
        "id": "g001",
        "name": "HR 组",
        "description": "人力资源部门",
        "member_count": 3,
        "kb_count": 2,
        "created_at": "2026-07-16T09:00:00"
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 10
  }
}

#### POST /api/user-groups - 创建用户组

请求体: { "name": "HR 组", "description": "人力资源部门" }
响应: {"code":0,"msg":"success","data":{"group_id":"g002"}}
异常: 400 - 用户组名称已存在

#### PUT /api/user-groups/{group_id} - 编辑用户组
请求体: { "name": "HR 组", "description": "人力资源部门" }
响应: {"code":0,"msg":"success","data":null}
异常: 404 - 用户组不存在

#### DELETE /api/user-groups/{group_id} - 删除用户组
级联删除组成员关联、知识库授权关联
响应: {"code":0,"msg":"success","data":null}

#### POST /api/user-groups/{group_id}/members - 添加组成员
请求体: { "user_ids": ["u002", "u003"] }
响应: {"code":0,"msg":"success","data":{"added_count":2}}
异常: 404 - 用户组不存在

#### DELETE /api/user-groups/{group_id}/members/{user_id} - 移除组成员
响应: {"code":0,"msg":"success","data":null}

#### POST /api/user-groups/{group_id}/kb-access - 授权知识库
请求体: { "kb_ids": ["kb001", "kb002"] }
响应: {"code":0,"msg":"success","data":{"authorized_count":2}}

#### DELETE /api/user-groups/{group_id}/kb-access/{kb_id} - 取消授权
响应: {"code":0,"msg":"success","data":null}

### 2.2 品牌配置（后端 B）

#### GET /api/system/branding - 获取品牌配置

鉴权: 公开接口，无需 Token

成功响应:
{
  "code": 0,
  "msg": "success",
  "data": {
    "system_name": "智能 RAG 平台",
    "logo_url": "",
    "theme_color": "#409EFF",
    "login_slogan": "AI 驱动的企业知识库",
    "footer_text": "Powered by RAG Platform"
  }
}

异常: 配置表为空时返回 code=4003，data 用默认值兜底

#### PUT /api/system/branding - 更新品牌配置

鉴权: 需 admin 权限

请求体: (字段均为可选，仅传入的字段会更新)
{
  "system_name": "XX 公司智能知识库",
  "logo_url": "/uploads/branding/logo.png",
  "theme_color": "#1890FF",
  "login_slogan": "XX 公司专属 AI 助手",
  "footer_text": "Copyright 2026 XX Company"
}

响应: {"code":0,"msg":"success","data":null}
异常: 403 - 非管理员无权修改

#### POST /api/system/branding/logo - 上传 Logo 文件

请求格式: multipart/form-data
参数: file (File, 必填, 仅支持 png/jpg/svg, 最大 2MB)
响应: {"code":0,"msg":"success","data":{"url":"/uploads/branding/logo_xxx.png"}}

## 三、V2 修改的现有接口

V2 追加字段（响应 items 中每个用户对象）:
- group_ids: ["g001", "g002"] - 所属用户组 ID 列表
- group_names: ["HR 组", "财务组"] - 所属用户组名称列表

### 3.2 POST/PUT /api/users - 新增/编辑用户（追加用户组绑定）
V2 追加字段（请求体）:
- group_ids: ["g001", "g002"] - 可选，同时写入 user_group_members

### 3.3 GET /api/knowledge-bases - 知识库列表（加权限过滤）
V2 变更: 非 admin 用户仅返回其授权用户组所关联的知识库列表

### 3.4 POST /api/rag/test_retrieve - 命中测试（加权限+状态校验）
V2 追加校验:
1. 校验请求用户对 kb_id 的访问权限。无权限 - 403
2. 校验 doc_id 对应文档状态是否为 completed。非 completed - 4002

### 3.5 POST /api/chat/send 和 POST /api/chat/stream - 对话（加权限+状态校验）
V2 追加校验:
1. 校验请求用户对 kb_id 的访问权限。无权限 - 403
2. 检验知识库下是否有至少一篇 completed 文档。全未就绪 - 4002

### 3.6 GET /api/knowledge-bases/{kb_id}/documents - 文档列表（加权限校验）
V2 追加: 访问前校验用户对 kb_id 是否有权限。无权限 - 403

## 四、V2 新增异常码

### 4.1 新增错误码

| code | 含义 | 触发场景 |
|:----:|------|------|
| 4001 | 用户组未授权 | 用户不在任何用户组中，或用户组未分配任何知识库 |
| 4002 | 文档未就绪 | 文档状态非 completed，不可用于检索 |
| 4003 | 白标配置缺失 | 系统品牌配置未初始化，使用默认值兜底 |

### 4.2 V2 降级策略速查表（补充条目）

| 故障场景 | 降级行为 | 前端提示 | 错误码 |
|------|------|------|:----:|
| 用户越权访问知识库 | 接口返回 403 | 您无权访问该资源 | 403 |
| 用户不在任何用户组 | 知识库列表返回空 | 账号未分配用户组 | 4001 |
| 文档未处理完成 | 禁止检索 | 文档正在处理中，请等待 | 4002 |
| 知识库下无可用文档 | 禁止对话 | 暂无已处理文档 | 4002 |
| 品牌配置表为空 | 使用默认值兜底 | 正常显示默认品牌 | 4003 |

### 4.3 V2 前端统一错误拦截（utils/request.js 追加）

Axios 响应拦截器新增处理:
- code=4002: 展示等待提示 ElMessage.warning
- code=403: 展示越权提示 ElMessage.error
- 其他异常处理保持不变

## 六、V2 新增后端工具模块契约

### 6.1 utils/permission.py - 权限中间件

函数签名:
- get_user_accessible_kb_ids(db, user_id) -> list[str]
  获取用户有权限访问的所有知识库 ID
  逻辑: admin 返回全部; 其他用户通过用户组关联查询

- require_kb_access(kb_id, user, db) -> bool
  校验当前用户对指定知识库是否有访问权限
  用于路由依赖注入，无权限时抛出 HTTPException(403)

- require_admin(user) -> bool
  校验当前用户是否为 admin
  非 admin 抛出 HTTPException(403)

### 6.2 utils/logger.py - 结构化日志

基于 loguru，统一 JSON 格式输出:
- 移除默认 handler
- 控制台 handler: 彩色输出，含级别/模块/行号
- 文件 handler: JSON 格式化，按天轮转，保留 7 天
- 每个请求自动注入 request_id 用于链路追踪

### 6.3 utils/metrics.py - Prometheus 自定义指标

自定义指标:
- rag_retrieve_total: Counter，检索请求总数
  标签: search_type, kb_id, status

- llm_call_total: Counter，LLM 调用总数
  标签: endpoint, status

- retrieve_latency: Histogram，检索延迟分布
  buckets: [0.1, 0.5, 1.0, 2.0, 5.0]

- llm_latency: Histogram，LLM 调用延迟分布
  buckets: [0.5, 1.0, 2.0, 5.0, 10.0]

- doc_status_gauge: Gauge，文档状态分布
  标签: status

### 6.4 rag_engine/rag_pipeline.py - RAG 全链路 + Langfuse 追踪

RAG 全链路管道接口:
- 接收 query + kb_id
- 检索（含权限+状态校验）
- LLM 生成
- 返回结果
- 全链路通过 Langfuse Trace 追踪
